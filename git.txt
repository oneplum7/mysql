需求 :  在linux的ubuntu终端上传编译器里面的代码到github
步骤：
一、将编译器里面的代码整合在ubuntu的一个目录。
         1、如果在ubuntu上需要新建一个目录（将项目整合到该目录下），终端操作命令是：mkdir  [目录名]
      
      其他知识点：
            （1）如果需要修改目录/文件名字，命令是：mv 【原目录名字】【目录新名字】
            （2）如果要查看文件内容并且想要进行修改，可以使用文本编辑器vim命令。命令是：vim  【文件】     例如：vim  mysql.py 
                              使用vim打开文件后：
                                      不改动文件情况下：
                                             退出vim命令:  shift  +  : +  q
                                       想要改动文件情况下：
                                                修改文件命令：i                  注解：就是直接按一个  i        没有任何别的添加  ，按过后就可以直接进行修改（增加文本，删除文本等等）。
                                                退出vim命令：
                                                           ①不保存文本修改就退出，命令是：   (冒号):   +    q !                  #注解：命令就是这两个     :q!
                                                           ②保存文件修改再退出，命令是：     （冒号):   +      wq             #注解：命令就是这三个     :wq

二、git命令使用
        1、切换到步骤一建立的目录下（也就是放项目的那个目录下），ubuntu切换目录命令：   cd 【目录名】
        
        2、（这一节和上传代码到git的步骤无关，因为我在linux过程中遇到该问题就记录一下）将某一文件复制到另外一个文件下命令： cp  [源文件名]   [新文件名]
                  #小提示：①    ./       表示当前目录
                                       ②    ../       表示上一层目录
                                       ③ 遇到不清楚的命令，用help命令：         
                                                                         例如：cp  --h
                                                                         例如： mv   --h
      
      3、终端切换到存放项目（也就是要上传到github的文件）的目录下之后，执行：git   init
          (执行回车后，会得到，已初始化空的 Git 仓库于 xxxxxx)

      4、完成3后，执行：git  add   ./
          (执行4后，可以输入命令  la   来看一下，是不是有 .git 这个文件，如果有就对了，一般正常情况是有的)           
          查看当前路径下面的所有文件：la
      
      5、完成4后，执行：git  commit   -m  "xxx"
      # "xxx" 是一些日志文件，可以理解成，你上传文件的提示信息
       #注解：后面这个地址在github上建好仓库后，会给我们一个地址
      #其实同时就在这个地址下面，也会给我们一个几行代码的文档，找到5这个命令，到linux执行就行
      #    ！！万一还是不会就help一下，例如：  git  commit  --help

      6、完成5后，执行：  git  remote  add origin  git@github,com:xxxx/xxx                      
      #注解：后面这个地址在github上建好仓库后，会给我们一个地址，就是给我们的这个地址
      #其实同时就在这个地址下面，也会给我们一个几行代码的文档，找到6这个命令，到linux执行就行

      7、完成6后，执行：  git  push   -u  origin  main 
      （以下记录，是执行到7这个步骤，报错了。报错之后怎么继续将文件上传到github。）
      （如果执行7成功，就去网上找接下来的步骤，不过快要创建成功了，加油！)
  ！！报错提示：error : src  refspec  main  does  not  match  any.
                                 error:  无法推送一些引用到  'git@github,com:xxxx/xxx '
      如果报同样的错误，就继续按下面步骤执行
     
      8、执行7后发现报错，执行：touch  README 
      #报错后步骤是采用网上一位前辈的解决方法（出处：https://www.jianshu.com/p/8d26730386f3），方便自己日后使用，也继续记录下来）

      9、执行8之后，执行： git  add  README

      10、完成9之后，执行：  git  commit  -m 'first commit'

      11、完成10后，执行： git  push  origin  master

      
      完成了，现在可以去github上刚刚建立的仓库里有没有上传成功的项目
